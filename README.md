# VK-бот для тренировки и интервью

Сервер на **Sinatra** принимает **Callback API** ВКонтакте (`POST /webhook`), сохраняет пользователей и вопросы в **PostgreSQL**, отвечает через **VK API**. Режим **«Живое интервью»** использует LLM (**DeepSeek** или **Yandex Cloud**).

## Возможности

- **Тренировка** — выбор темы (категории вроде Backend / Frontend), вопросы с подсказкой и показом ответа, оценка ответа, переход «Дальше».
- **Загрузить вопросы** — ручной ввод вопроса/ответа/подсказки или загрузка файла (документ ВК).
- **Живое интервью** — диалог с «интервьюером» на базе LLM, команда **«Стоп»** для выхода.
- **Статистика** — сводка по живому интервью.

## Требования

- [Docker](https://docs.docker.com/get-docker/) и Docker Compose **или** Ruby **3.3**, Bundler, PostgreSQL 16+.
- Сообщество ВК с включённым **Callback API** и доступом к интернету до вашего URL вебхука.

## Переменные окружения

Создайте в корне проекта файл **`.env`** (для `docker compose` он подхватывается автоматически, см. [`docker-compose.yml`](docker-compose.yml)).

### Обязательные для бота

| Переменная | Описание |
|------------|----------|
| `VK_ACCESS_TOKEN` | Токен сообщества с правами на сообщения (например из «Работа с API» → ключ доступа). |
| `VK_GROUP_ID` | ID группы (число без минуса или как в настройках API). |
| `VK_CONFIRMATION_CODE` | Строка подтверждения из настроек Callback API. |

### База данных

- В Docker переменная **`DATABASE_URL`** задаётся в `docker-compose.yml` (подключение к сервису `postgres`).
- Локально без Docker можно задать, например:  
  `DATABASE_URL=postgres://postgres:postgres@127.0.0.1:5432/vk_bot`

### LLM для «Живого интервью»

По умолчанию используется **DeepSeek**:

| Переменная | Описание |
|------------|----------|
| `DEEPSEEK_API_KEY` | Ключ API DeepSeek. |

Чтобы использовать **Yandex Cloud**:

| Переменная | Описание |
|------------|----------|
| `LLM_PROVIDER` | Значение `yandex` |
| `YANDEX_CLOUD_API_KEY` | API-ключ сервисного аккаунта. |
| `YANDEX_CLOUD_FOLDER_ID` | ID каталога. |
| `YANDEX_CLOUD_MODEL` | Имя модели (как в Yandex AI Studio). |

Опционально (есть разумные значения по умолчанию в коде):  
`LLM_TIMEOUT_SECONDS`, `LLM_REQUEST_RETRIES`, `INTERVIEW_HISTORY_ITEMS`, `LLM_MAX_OUTPUT_TOKENS`, `INTERVIEW_EVAL_MAX_OUTPUT_TOKENS`, `LLM_INPUT_MAX_CHARS`.

### Пример минимального `.env`

```env
VK_ACCESS_TOKEN=vk1.a...
VK_GROUP_ID=123456789
VK_CONFIRMATION_CODE=abcdef12

DEEPSEEK_API_KEY=sk-...
```

Для Docker строку `DATABASE_URL` в `.env` дублировать не нужно — её подставляет Compose (если не переопределите сами).

## Запуск через Docker Compose

Из корня репозитория:

```bash
docker compose up --build
```

- HTTP-сервер слушает порт **4567** (`http://localhost:4567`).
- Проверка: откройте в браузере `http://localhost:4567/` — должно отобразиться `VK bot webhook is running`.
- PostgreSQL доступен на хосте на порту **5432** (данные в volume `pg_data`).

Остановка: `Ctrl+C` или `docker compose down`. Полный сброс данных БД: `docker compose down -v`.

## Настройка Callback API ВКонтакте

1. Управление сообществом → **Настройки** → **Работа с API** → **Callback API**.
2. **Адрес сервера**: `https://<ваш-домен>/webhook` (для разработки удобен туннель, например ngrok; в [`app.rb`](app.rb) в `permitted_hosts` уже указан суффикс `.tuna.am` — добавьте свой хост при необходимости).
3. Версия API — совместимая с кодом (в клиенте VK используется **5.199**, см. [`apis/vk.rb`](apis/vk.rb)).
4. Подпишитесь на событие **«Входящее сообщение»** (`message_new`).
5. При первом запросе VK пришлёт **подтверждение** — сервер ответит строкой из `VK_CONFIRMATION_CODE`.

## Локальный запуск без Docker

```bash
bundle install
# поднимите PostgreSQL и создайте БД vk_bot (или свою), затем:
export DATABASE_URL=postgres://postgres:postgres@127.0.0.1:5432/vk_bot
# заполните .env или экспортируйте VK_* и DEEPSEEK_* / YANDEX_*
bundle exec rackup config.ru -o 0.0.0.0 -p 4567
```

При старте выполняется миграция и сиды через [`Store.bootstrap!`](store.rb) (см. [`db/migrations/001_init.sql`](db/migrations/001_init.sql), [`db/seeds.sql`](db/seeds.sql)).

## Примеры использования в переписке с ботом

Напишите боту в сообщения сообщества (от имени пользователя). Команды и кнопки совпадают с подписями в [`bot.rb`](bot.rb).

### Главное меню

- **«Начать»** или **«/start»** — главное меню с кнопками: Тренировка, Загрузить вопросы, Живое интервью, Статистика.
- **«Меню»** — вернуться в главное меню из большинства сценариев.

### Тренировка

1. Нажмите **«Тренировка»** (или отправьте текст «Тренировка»).
2. Выберите тему, например **«Backend»**.
3. Ответьте на вопрос текстом.
4. Можно нажать **«Подсказка»**, **«Показать ответ»**, затем **«Дальше»** для следующего вопроса в категории.

### Загрузка своих вопросов

1. **«Загрузить вопросы»** → **«Вручную»** (или загрузка файла-документа по подсказкам бота).
2. Укажите категорию (например **Backend**), затем текст вопроса, правильный ответ и подсказку (или `-` без подсказки).
3. После сохранения вопрос доступен в тренировке.

### Живое интервью

1. **«Живое интервью»** → выберите тему (**Backend**, **Рандом** и т.д.).
2. Отвечайте на вопросы сообщениями; бот оценивает ответ и задаёт следующий вопрос.
3. **«Стоп»** — завершить интервью.

### Статистика

- **«Статистика»** — сводка по живому интервью (после того как были ответы в этом режиме).

## Разработка и проверки

```bash
bundle exec rubocop
bundle exec ruby tasks/smoke_mock.rb
```

Последняя команда прогоняет сценарий с подменой Store и API на заглушки и в конце печатает `SMOKE_OK` (удобно для CI).
